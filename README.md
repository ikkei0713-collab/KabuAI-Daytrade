# KabuAI デイトレード

**AI自律学習型の日本株デイトレードシステム。**
立花証券APIで実取引を行い、毎トレードからデータを蓄積し、戦略を自動で進化させ続ける。
目的はただ一つ——**資金を増やすこと。**

## コンセプト

```
トレード → データ蓄積 → 分析 → 戦略改善 → より良いトレード → ...
```

AIが24時間思考し続け、過去の全トレードから学び、戦略を自動調整する。
人間が寝ている間もデータは貯まり、次の寄り付きには昨日より賢くなっている。

## v6.0 プロ仕様デイトレードエンジン (2026-04-02)

立花証券API + Yahoo Finance + J-Quants + TDnet + Telegram + OpenAI。

### 日々の運用フロー（完全自動）

```
【大引け後 15:00】
  → 大引けレポート (Telegram通知)
  → 翌日スクリーニング (4,400銘柄→上位20銘柄→JSON保存)

【翌朝 9:00】
  → セッション自動復元 (電話認証不要なら即起動)
  → 前日スクリーニング結果を読み込み
  → 30秒ごとにスキャン→シグナル→自動発注

【場中 9:00-15:00】
  → ポジション監視 (30秒ごと、トレーリングストップ付き)
  → 未約定注文管理 (1分ごと、古い指値は自動取消)
  → TDnet適時開示 (5分ごとスキャン)
  → Yahoo Finance 1分足 (2分ごと更新)

【前場引け 11:30】
  → セッションレポート (Telegram通知)

【14:50】
  → 全ポジション強制決済

【いつでも】
  → Telegramで話しかけると状況を教えてくれる (AI対話)
```

### 実装済み機能

| 機能 | 説明 |
|------|------|
| **Telegram AI対話** | フリーテキストで話しかけるとGPT-4o-miniがトレード状況を踏まえて回答 |
| **Telegram通知** | エントリー/決済/前場引け/大引けレポート/スクリーニング完了を自動通知 |
| **大引け後スクリーニング** | 翌日の投資候補を自動選定→JSON保存→翌朝読み込み |
| **未約定注文管理** | 1分ごとにチェック、古い指値注文は自動取消 |
| **Yahoo Finance 1分足** | 全銘柄のリアルタイム価格+日中足を2分ごと取得 |
| **マルチソース価格** | 立花API → Yahoo Finance → 分足 → 日足の4段階フォールバック |
| **TDnet適時開示** | 5分ごとにスキャン、上方修正・自社株買い等を即座に戦略に注入 |
| **動的銘柄発見** | J-Quantsから余力で買える低位株を自動発見（4,400銘柄から絞り込み） |
| **トレーリングストップ** | 含み益が1ATR超えたら最高値から自動追従 |
| **時間帯フィルター** | ORBは9:00-9:45、open_driveは9:00-9:30のみ等 |
| **利確は指値** | 利確=指値（有利に約定）、損切り=成行（確実に逃げる） |
| **セッション保存** | 再起動時にログインセッションを自動復元 |
| **ポジション自動同期** | APIから保有銘柄・残高を自動取得 |
| **14:50強制決済** | 引け前に全ポジション自動決済 |
| **銘柄名表示** | 4,464銘柄の名称マップでコードだけでなく会社名も表示 |
| **成行エントリー** | 即約定で機会損失を防止（利確は指値、損切りは成行） |
| **未約定自動取消** | 起動時に前回の未約定注文を自動取消して余力を解放 |
| **セッション切れ通知** | Telegramに電話番号付きで通知、タップで発信可能 |
| **Event Intelligence** | TDnet+価格アクションからイベント重要度・波及スコアを自前計算 |
| **時間減衰** | 古いパターンの影響を10営業日半減期で薄める |
| **シグナル重複排除** | 同一銘柄60秒以内の重複スキャンを防止 |
| **MMR信号選択** | 同セクター銘柄への集中投資をλ=0.7で回避 |
| **日中VWAP** | Yahoo Finance 1分足からリアルタイムVWAPを計算 |
| **レジーム最適化** | 低位株のvolatile誤判定を修正、トレード機会を確保 |
| **IDTWパターンマッチング** | 中川(2019)に基づくIndexing DTWで直近20日の値動きを過去と照合、confidence boost |
| **レジーム×戦略フィルター** | バックテストOOS PF≥1.2の勝ち組み合わせのみ許可 |
| **約定価格ベース通知** | 立花APIから実約定価格を取得し正確なエントリー通知 |

### 安全設計

| 項目 | 設定 |
|------|------|
| 日次最大損失 | **-¥3,000** で全ポジション決済・停止 |
| 同時保有上限 | **2ポジション** |
| 1注文上限 | 資金の **40%** まで |
| 強制決済 | **14:50** に全ポジション成行売り |
| 戦略auto off | PF < 0.90 or 勝率 < 40% で自動停止 |

```

│              毎日 15:30 (引け後)                    │
│  1. 日次ナレッジ抽出                                │
│     - 勝ち/負けパターンの自動マイニング              │
│     - 特徴量 × 損益の相関分析                       │
│     - 戦略 × レジーム × 銘柄の相性マトリクス更新      │
│  2. 戦略auto_toggle                                │
│     - ローリング成績でON/OFF自動制御                 │
│  3. フィードバックパッケージ生成                      │

└──────────┬───────────────────────────────────────┘
           ▼
┌──────────────────────────────────────────────────┐
│              バックグラウンド (随時)                  │
│  - run_backtest_learn.py: 過去データで全戦略検証      │
│  - run_optimize.py: パラメータ空間ランダムサーチ       │
│  - OOS (Out-of-Sample) で過学習を検出                │
│  - 最適パラメータをJSON保存→次回起動時に自動適用       │
└──────────┬───────────────────────────────────────┘
           ▼
┌──────────────────────────────────────────────────┐
│              翌朝 09:00 (寄り付き)                   │
│  昨日のデータで更新された戦略でトレード開始            │
│  → 新しいデータが貯まる → ループ継続                  │
└──────────────────────────────────────────────────┘
```

### データ蓄積で強くなる理由

1. **トレードするほどデータが増える** — 勝ちパターン・負けパターンの統計的信頼度が上がる
2. **レジーム × 戦略の相性が見える** — 「この地合いでこの戦略は勝てない」を数字で判断
3. **銘柄相性を学習** — strategy × ticker の勝率をDB蓄積、同じ失敗を繰り返さない
4. **パラメータが自動最適化** — OOS検証で過学習を防ぎつつ、利益最大化のパラメータを探索
5. **特徴量の有効性を定量化** — IC/ICIRで「どの指標が本当に効いているか」を継続追跡

## Event Intelligence（自前イベント分析エンジン）

外部APIに依存せず、TDnet + J-Quants + 価格アクションから**イベント重要度・波及スコア・企業特徴**を自前計算する。

- **イベント重要度 (event_importance_score)**: TDnet開示タイプ+キーワードから0-1で算出（上方修正=0.9, 自社株買い=0.7等）
- **鮮度スコア (event_freshness_score)**: 開示からの経過時間で減衰（1時間以内=1.0, 6時間=0.7, 24時間超=0.1）
- **波及スコア (propagation_score)**: 同業種グループ内の他銘柄イベントから波及効果を推定
- **企業特徴 (company_feature_score)**: 出来高比率・ATR%・価格帯から算出
- **価格アクション検出**: TDnet開示がなくても急騰(+3%)+出来高急増(2倍)を自動検出
- **watchlist加点**: importance×0.10 + freshness×0.05 + propagation×0.07 + company×0.05
- **confidence boost**: vwap_reclaimに高重要度+0.07, 波及+0.04, stale-0.08

## クイックスタート

```bash
# セットアップ
cd ~/dev/KabuAI-Day
python3.13 -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
cp .env.example .env  # API キーを記入

# 実取引ボット起動（電話認証 → 3分以内に実行）
.venv/bin/python run_live_trade.py

# バックテスト学習（バックグラウンド）
nohup .venv/bin/python run_backtest_learn.py >> logs/backtest.log 2>&1 &

# パラメータ最適化
nohup .venv/bin/python run_optimize.py >> logs/optimize.log 2>&1 &

# ログ確認
tail -f logs/live_trade.log
```
